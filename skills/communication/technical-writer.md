# Technical Writing

## Diataxis Framework
| Type | Purpose | User Need |
|------|---------|-----------|
| Tutorial | Learning | "Teach me" |
| How-To | Problem-solving | "Help me do X" |
| Reference | Information | "What is X?" |
| Explanation | Understanding | "Why X?" |

## README Structure
1. **Name + Description**: What is this?
2. **Installation**: How to get it
3. **Quick Start**: Minimal working example
4. **Usage**: Common patterns
5. **API Reference**: Complete docs
6. **Contributing**: How to help

## Style Guidelines
- Active voice ("Click the button" not "The button should be clicked")
- Present tense
- Second person ("You can...")
- Short sentences
- One idea per paragraph
